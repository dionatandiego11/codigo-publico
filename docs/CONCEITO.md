# Código Público — Conceito

O **Código Público** é uma infraestrutura pública de **Orçamento Participativo municipal**.

Ele usa a lógica de software versionado como camada interna de governança: histórico, estado, auditoria, filtros, merge institucional e release. Mas a experiência principal não é “GitHub de leis”. Para o cidadão, o sistema começa no território e no problema concreto.

> O Código Público transforma uma demanda simples de um território em decisão orçamentária rastreável.

## Mapa conceitual

```txt
Município                 = instância política do sistema
Bairro/comunidade/distrito = território-base
Cidadão                  = participante vinculado a um território
Demanda                  = problema público simples
Proposta                 = demanda amadurecida em solução possível
Fork                     = alternativa para o mesmo problema
Filtro territorial       = organização local da esteira
Circuit breaker          = proteção jurídico-orçamentária
Votação territorial      = priorização cidadã
Matriz do OP             = consolidação municipal das prioridades
Maintainer Geral         = Legislativo municipal no rito do OP
Maintainer Técnico       = operador de infraestrutura
Maintainer Territorial   = representante sorteado do território
Merge institucional      = incorporação no rito formal
Release do ciclo         = versão pública consolidada do OP
Execução                 = acompanhamento do item aprovado
Blockchain               = prova externa de integridade, sem dado pessoal
```

## O ciclo

```txt
Ciclo do OP aberto
  → cadastro e vínculo territorial
  → inscrição para Maintainer Territorial
  → sorteio auditável
  → demanda simples
  → apoio ou não apoio comunitário
  → agrupamento ou fork
  → maturação territorial
  → filtro territorial
  → circuit breaker jurídico-orçamentário
  → demanda apta
  → votação territorial
  → consolidação municipal
  → matriz do OP
  → institucionalização na Câmara
  → merge no PPA/LDO/LOA
  → release do ciclo
  → execução fiscalizada
  → aprendizado do próximo ciclo
```

## O cidadão começa pelo problema

O cidadão não precisa saber escrever lei, emenda ou parecer técnico.

Ele pode começar com algo simples:

> Falta médico no PSF do meu bairro.

A esteira do sistema transforma esse registro em algo mais maduro:

- outras pessoas apoiam, contestam ou complementam;
- demandas parecidas são agrupadas;
- soluções diferentes viram forks;
- filtros públicos indicam se há problema jurídico, territorial ou orçamentário;
- propostas aptas vão para votação territorial;
- prioridades entram na matriz municipal do OP;
- o Legislativo incorpora o que avançar ao rito de PPA, LDO e LOA;
- a execução vira item fiscalizável.

## Regra comum e regra local

O projeto opera com duas camadas.

```txt
Kernel comum
  regras democráticas mínimas, privacidade, auditoria, sorteio, retornos da esteira

Regimento local
  parâmetros do município: calendário, mandato, prazos, quórum, envelope e índice de carência
```

O município pode parametrizar o processo, mas não pode violar o núcleo comum: território sem voz, maintainer vitalício, filtro sem justificativa, exposição de voto individual ou dado sensível em blockchain.

## Frases-síntese

> A justiça social do OP começa pela justiça territorial.

> A demanda nasce simples e ganha maturidade por camadas.

> Toda negativa deve gerar justificativa, retorno, fork, recurso ou memória pública.

> O Legislativo é o Maintainer Geral do rito, mas não pode ser veto invisível.

> A execução ou frustração de uma promessa precisa influenciar o próximo ciclo.
